## 🧠 DEEP DESIGN THINKING (MANDATÓRIO)

**NUNCA comece a codar sem antes completar esta análise interna:**

### 1. Scan de Clichês Modernos (Anti-Safe Harbor)
- Estou usando o "Standard Hero Split" (Texto esquerda / Imagem direita)? → **TRAIA ISSO.**
- Estou usando Bento Grids por preguiça de criar um layout? → **QUEBRE O GRID.**
- Estou usando as cores seguras de SaaS (Azul/Ciano)? → **DISRUPTE A PALETA.**

### 2. Hipótese Topológica
Escolha um caminho radical e comprometa-se:
- **FRAGMENTAÇÃO**: Quebre a página em camadas sobrepostas sem lógica vertical rígida.
- **BRUTALISMO TIPOGRÁFICO**: Texto é 80% do peso visual; imagens são artefatos escondidos.
- **TENSÃO ASSIMÉTRICA (90/10)**: Force um conflito visual empurrando tudo para uma borda extrema.
- **NARRATIVA CONTÍNUA**: Sem seções, apenas um fluxo narrativo de fragmentos.

## 🛑 REGRAS DE OURO

1. **BANIMENTO DO ROXO**: Nunca use roxo/magenta como cor primária (clichê de IA), a menos que solicitado.
2. **GEOMETRIA EXTREMA**: Evite o "conforto" de 4px-8px de border-radius. Use **0px-2px** (Agressivo/Técnico) ou **32px+** (Orgânico/Social).
3. **ANIMAÇÃO É OBRIGATÓRIA**: UI estática é falha. Use reveal animations e micro-interações com física de mola.
4. **PERFORMANCE EM PRIMEIRO LUGAR**: Use Server Components por padrão. Meça antes de otimizar.
