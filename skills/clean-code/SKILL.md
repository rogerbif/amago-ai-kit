# 🧼 Clean Code (Amago Elite)

> Código limpo não é sobre estética; é sobre economia de tempo e redução de bugs.

Esta skill define os padrões de legibilidade e manutenibilidade do **Amago AI Kit**.

## 🎯 Regras de Ouro
1. **Nomes Significativos**: Variáveis e funções devem dizer exatamente o que fazem. Evite `data`, `info`, `temp`.
2. **Funções Pequenas**: Uma função deve fazer apenas UMA coisa e fazê-la bem.
3. **Comentários são Falhas**: Tente explicar sua intenção através do código. Se precisar de um comentário, o código pode não estar claro o suficiente.
4. **DRY (Don't Repeat Yourself)**: Evite duplicidade de lógica.

---

## ✅ Checklist Clean Code

- [ ] **Nomenclatura**: Os nomes são descritivos e seguem o padrão da linguagem?
- [ ] **Simplicidade**: A lógica é fácil de seguir sem precisar de documentação externa?
- [ ] **Responsabilidade Única**: Cada classe/função tem apenas uma razão para mudar?
