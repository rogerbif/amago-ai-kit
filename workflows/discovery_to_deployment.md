# Amago AI Kit: Workflow de Ciclo de Vida Completo

Este fluxo define as etapas padronizadas para transformar uma ideia em um produto final: garantindo qualidade e segurança em cada fase.

## Fase 1: Descoberta e Planejamento (Persona: PM)

1. Reunião de Briefing: Entender o pedido do usuário e coletar requisitos iniciais.
2. Criação do Plano: Elaborar o documento "PLAN.md" com a decomposição das tarefas.
3. Validação do Usuário: Obter aprovação do plano antes de iniciar qualquer código.

## Fase 2: Arquitetura e Segurança (Personas: Backend e Security)

1. Design do Sistema: Definir a estrutura de dados e os padrões de API (Backend).
2. Auditoria de Segurança Inicial: Validar o design contra vulnerabilidades conhecidas (Security).
3. Configuração de Infraestrutura: Preparar o ambiente de desenvolvimento.

## Fase 3: Desenvolvimento e Interface (Personas: Frontend e Backend)

1. Implementação de Backend: Construir a lógica de negócio e persistência.
2. Implementação de Frontend: Criar a interface seguindo os padrões de design premium.
3. Integração: Conectar o frontend ao backend utilizando as melhores práticas de Modern ES.

## Fase 4: Garantia de Qualidade e Deploy (Personas: QA e Manager)

1. Automação de Testes: Criar e executar testes unitários, de integração e E2E (QA).
2. Auditoria Final: Executar scripts de segurança e linting para garantir conformidade técnica.
3. Entrega Final: Realizar o deploy e apresentar o resultado ao usuário para aprovação final.

## Regras do Fluxo

1. Nenhum código deve ser escrito sem um plano aprovado.
2. A segurança deve ser consultada em todas as decisões arquiteturais.
3. O QA deve validar todos os critérios de aceitação definidos pelo PM.
