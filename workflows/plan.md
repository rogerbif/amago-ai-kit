# ?? Fluxo de Planejamento

> Planejar é economizar tempo de debug.

Siga este fluxo sempre que iniciar uma tarefa complexa de implementação ou refatoração.

## 1. Fase de Descoberta (Discovery)
- **O quê**: O que exatamente o usuário quer?
- **Por quê**: Qual o valor de negócio ou técnico desta mudança?
- **Restrições**: Existem limitações de stack, tempo ou design?

## 2. Fase de Planejamento Técnico
- **Arquitetura**: Como essa mudança se encaixa no sistema atual?
- **Impacto**: Quais arquivos ou módulos serão afetados? (Breaking changes?)
- **ADR**: Se for uma decisão arquitetural permanente, crie um ADR.

## 3. Elaboração do Plano
- Crie o `implementation_plan.md`.
- Quebre a tarefa em sub-tarefas granulares.
- Defina os critérios de aceitação para cada uma.

## 4. Validação Socrática
- Faça pelo menos 3 perguntas ao usuário sobre casos de borda ou trade-offs antes de começar a codar.
